Kalispel Tribe - Utility Asset Management

Project Overview

This comprehensive GIS project involved creating an inventory of accessible utility assets throughout Kalispel Tribal Lands. In collaboration with the Kalispel Tribe’s Planning, Transportation, Public Works, Maintenance, and Economic Development teams, the project focused on creating a GIS-based digital inventory by digitizing decades of as-builts and hand drawings and field data collection of current conditions with Field Maps. This resulted in a structured geodatabase housing Tribal assets from multiple sources and formats. The intent was to create a baseline inventory from which to build upon and continue future updates and development.

Technical Implementation

I utilized ArcGIS Pro for data management and analysis, with a custom web application built using Experience Builder. Field technicians used Field Maps for data collection and updates. For the stormwater dataset, I used the ESRI Stormwater Data Management solution schema. The Tribe controls and manages their stormwater system so a more comprehensive data model was optimal for that element. I created a simpler schema for other utilities not under Tribal control such as water, sewer, electric and telecom.

The main reservation inventory was captured from existing as-builts and field data collection while the Airway Heights area was created by converting CAD .dwg files to a geodatabase. I created a python script to help automate the translation of limited CAD attribute data to the resulting schemas I created.

Shoalwater Bay Tribe - Road Maintenance Plan

Project Overview

For this project, I developed a comprehensive GIS data-driven road maintenance plan for the Shoalwater Bay Tribe. The system provides detailed cost estimates for each maintenance activity, enabling efficient budget planning and resource allocation for tribal transportation infrastructure. Previously, maintenance plan data was manually processed and could take weeks to generate a final written report. With the GIS data-driven method I created, a web app and report can be completed in days instead of weeks.

Key Features

  • Detailed road condition assessment and rating system
  • Cost estimation models for various maintenance activities
  • Priority ranking system for maintenance scheduling
  • Integration with tribal budget planning processes
  • Interactive Experience Builder app for display and decision-making

Technical Implementation

At my direction, a field technician drove all the roads to be covered in the maintenance plan and conducted a conditions assessment survey using Field Maps. I created a schema to capture maintenance need and priority while enabling attachments so the field tech could capture photos for each maintenance need.

In post-processing, I created cost fields with attribute rules written in Arcade for each maintenance need. The rules calculate cost by multiplying the WSDOT unit cost per item by length or area of maintenance need. By calculating cost directly in the data, it eliminates the time and expense of engineers estimates. Where the engineer would be using the same formulas manually, the GIS does the estimating automatically.

In addition to the Experience Builder app, the deliverables include a written report and summary table. I created a python script to summarize the data by maintenance need, priority and cost per route section, then generate an excel table (and PDF) with visually appealing formatting to include in the final report.

Confederated Salish and Kootenai Tribes - Road Inventory (NTTFI)

Project Overview

This project involved updating the National Tribal Transportation Facility Inventory (NTTFI) for the Confederated Salish and Kootenai Tribes of the Flathead Reservation in Montana as part of their Tribal Transportation Program. The work ensured compliance with federal requirements while providing the tribe with accurate transportation infrastructure data.

Key Features

  • Complete road network inventory following NTTFI standards
  • GPS-accurate road centerline mapping
  • Attribute data collection for all transportation facilities
  • Integration with federal reporting requirements
  • Quality assurance and validation processes

Technical Implementation

The project utilized ArcGIS Pro for data collection and management, with custom tools developed to ensure NTTFI compliance. Field verification was conducted using high-accuracy Trimble GPS equipment with data collection in Field Maps. All data was validated through rigorous QA/QC processes before submission to federal agencies.
